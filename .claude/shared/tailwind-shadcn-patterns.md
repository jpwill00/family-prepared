# Tailwind + shadcn/ui Patterns

## shadcn/ui Component Usage

Use shadcn/ui primitives from `src/components/ui/` for all interactive elements:

```tsx
// Good — uses Radix-backed shadcn primitive
import { Button } from "@/components/ui/button";
import { Dialog, DialogContent, DialogHeader } from "@/components/ui/dialog";

// Bad — raw HTML button without accessible role/keyboard handling
<button onClick={...}>Save</button>
```

Never edit files inside `src/components/ui/` — they are generated by `shadcn add` and will be overwritten. Put customizations in wrapper components.

## Adding New shadcn Components

```bash
pnpm dlx shadcn@latest add <component-name>
```

Then import from `@/components/ui/<component-name>`.

## Tailwind Class Ordering

Follow the Prettier Tailwind plugin order (enforced by `eslint-plugin-tailwindcss`):
1. Layout (display, position, flex/grid)
2. Spacing (margin, padding)
3. Sizing (width, height)
4. Typography
5. Colors and backgrounds
6. Borders
7. Effects
8. Responsive prefixes last

## Zone Color Coding (Build Plan Convention)

The four zones have semantic colors — use them consistently in UI:

| Zone | Color | Tailwind |
|------|-------|---------|
| `plan/` | Green | `text-green-700`, `bg-green-50` |
| `library/` | Blue | `text-blue-700`, `bg-blue-50` |
| `packs/` | Purple | `text-purple-700`, `bg-purple-50` |
| `custom/` | Yellow/Amber | `text-amber-700`, `bg-amber-50` |

## Accessibility Rules

- All interactive elements must have visible focus rings (`focus-visible:ring-2`)
- Icon-only buttons need `aria-label`
- Form inputs need associated `<label>` (use shadcn `<Label>` + `htmlFor`)
- Color is never the only differentiator — pair with text or icons

```tsx
// Good — accessible icon button
<Button variant="ghost" size="icon" aria-label="Delete member">
  <Trash2 className="h-4 w-4" />
</Button>

// Bad — no accessible label
<Button variant="ghost" size="icon">
  <Trash2 className="h-4 w-4" />
</Button>
```

## Responsive Design

Mobile-first. The app targets non-technical users who may primarily use phones:
- Default styles = mobile
- `md:` prefix = tablet+
- `lg:` prefix = desktop

## Dark Mode
