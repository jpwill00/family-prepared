# PWA + Service Worker Rules

## Registration Rule — CRITICAL

**Service worker registration happens only in `src/main.tsx`.**

```ts
// main.tsx — ONLY place SW registration lives
import { registerSW } from "virtual:pwa-register";
const updateSW = registerSW({ onNeedRefresh() { ... } });
```

Never register or interact with the SW from any other file.

## vite-plugin-pwa Configuration

The SW is generated by `vite-plugin-pwa` (Workbox under the hood). Config lives in `vite.config.ts`:

```ts
VitePWA({
  registerType: "autoUpdate",
  workbox: {
    globPatterns: ["**/*.{js,css,html,ico,png,svg,woff2}"],
    runtimeCaching: [
      // Cache Leaflet tiles for offline map use
      {
        urlPattern: /^https:\/\/tile\.openstreetmap\.org\/.*/i,
        handler: "CacheFirst",
        options: {
          cacheName: "osm-tiles",
          expiration: { maxEntries: 200, maxAgeSeconds: 60 * 60 * 24 * 30 },
        },
      },
    ],
  },
  manifest: {
    name: "Family Prepared",
    short_name: "FamPrep",
    description: "Offline emergency preparedness plans for your family",
    theme_color: "#16a34a",
    display: "standalone",
    icons: [
      { src: "icon-192.png", sizes: "192x192", type: "image/png" },
      { src: "icon-512.png", sizes: "512x512", type: "image/png" },
    ],
  },
})
```

## Dev vs Prod

- SW is **not active in `pnpm dev`** — Vite HMR and SW conflict
- To test PWA behavior: `pnpm build && pnpm preview`
- To test install prompt: open the preview URL in Chrome, look for the address bar install icon

## Offline Requirements

The app must be **fully functional offline**:
- All app shell assets cached by Workbox on first install
- All user data lives in IndexedDB (survives offline)
- Routes that need GitHub sync show a "working offline" notice but don't break
- Leaflet map tiles cached via `CacheFirst` strategy

## Update Flow

`registerType: "autoUpdate"` means the SW updates silently in the background. On next page load the user gets the new version. If you need a manual "Update available — reload" prompt, switch to `"prompt"` and wire up the `onNeedRefresh` callback to a toast.

## What NOT to Do

```ts
// ❌ Never do this outside main.tsx
navigator.serviceWorker.register("/sw.js");

// ❌ Never cache sensitive data in the SW cache
// All user data lives in IndexedDB, not the Cache API

// ❌ Never use SW to intercept GitHub API calls for auth
// Auth lives in lib/github/auth.ts and uses idb-keyval for token storage
```
